いよいよWebサイトを作りたいと思います。

SharePoint Server 2007では、Webサイトのことをサイトと呼んでいます。
サイトを作る前に、Webアプリケーションを作る必要があります。
SharePoint Server 2007ではWebアプリケーションがサイトを束ねるひとつの括りになっていて、ひとつのWebアプリケーションの中に複数のサイトを立てることで、これらのサイトを一括で管理することができるようになっています。

Webアプリケーションとサイトの関係は、そのままURLにも反映されていて、たとえばWebアプリケーションを「app」という名前で作成し、このWebアプリケーションに所属するサイトを「site1」、「site2」という名前で作成すると、それぞれのサイトのURLは以下のようになります。
http://app/site1、http://app/site2
と、いうことで、サイトを作るためにはまずWebアプリケーションを作る必要があります。

では、以下にWebアプリケーションを作る手順を書きます。
なお、今回は、http://labserver/ というURLでサイトを作ります。ポートは80を使用します。
http://labserver/site1 のように、ルートの下にサイトを作るのではなく、今回のようにルートにサイトを作ることもできます。

**１．サーバーの全体管理ページを表示します。****２．サーバーのサービスページへ移動します。**
[サーバー構成の管理]タブの[トポロジおよびサービス]セクションの[サーバーのサービス]をクリックしてください。
**３．Windows SharePoint Services Web Applicationサービスを起動。**
[以下の表のサービスを起動]と書いてあるリストの中から、「Windows SharePoint Services Web Application」と書いてあるサービスを見つけ、[開始]リンクをクリック。
しばらくするとサービスが開始します。
**４．アプリケーション構成の管理ページへ移動します。****５．Webアプリケーションの作成または拡張ページを表示します。**
サーバーの全体管理ページにアクセスし、[アプリケーションの構成]タブの[SharePoint Web アプリケーション構成の管理]セクションにある、[Web アプリケーションの作成または拡張]をクリックします。
**６．[新しい Web アプリケーションの作成]をクリックします。****７．新しいWebアプリケーションを作成します。**
色々と設定項目がありますが、以下の設定を除いてすべてデフォルトで行きます。
アプリケーションプールセクションの[ユーザー名]と[パスワード]には、構成データベースに接続可能なユーザーを指定してください。
今回はadministratorを指定します。
入力が完了したら、[OK]ボタンをクリックします。
すると、処理中ページに遷移します。
**８．Webアプリケーション作成完了。**
処理が完了してこのページが表示されたら、Webアプリケーションの完成です。
続いてトップレベルサイトを作成します。
文中にある[サイト コレクションの作成]というリンクをクリックしてください。
**９．トップレベルサイト作成。**
サイトコレクションの作成ページの各設定項目を入力します。
[タイトル]と[説明]は自由に入力してください。
ここで入力した文字は、サイトの左上のところに表示されます。
[テンプレート]は自由に選択してください。
[サイトコレクションの管理者]は、存在するユーザーの名前を入力してください。

それ以外の設定項目は、今回はデフォルトのまま行きます。 
すべて入力が完了したら、[OK]ボタンをクリックします。
すると、処理中ページに遷移します。
**１０．トップレベルサイト作成完了。**
処理が完了してこのページが表示されたら、サイトの完成です。

リンクをクリックしてサイトにアクセスしてみてください。
初回表示時はコンパイルなどが行われるため、表示までに少々時間がかかります。
しばらく経つとたった今作成した、できたてホヤホヤのサイトが表示されます！

お疲れ様でした！
